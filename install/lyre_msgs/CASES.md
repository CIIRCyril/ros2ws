# 用例文档

## 前置说明

### 安装包命名规范

```
<workspace>-<platform>-<version>-<timestamp>.tar.gz
```

字段说明：

- workspace

  固定为 lyre_ros2

- platform

  x86: x86_64

  orin: aarch64

- version

  v2.x.x

- timestamp

  yyyyMMdd-HHmmss

示例名称：

```
lyre_ros2-aarch64-v2.0.1-20250721_192021.tar.gz
```

### 安装环境说明

### 检查目标主控板

对于 Lite 版本的设备，因为机器人没有 Orin 主控板，应该安装 x86_64 版本。

对于其它版本的设备，需要将程序包安装在 Orin 主控板，应该安装 aarch64 版本。

### 检查 3588 语音板

对于没有 3588 语音板的机器，可以在装有喇叭的主控板上安装程序包，以实现音频播放功能。

对于有 3588 语音板的机器，可以在和 3588 通过网线直连的主控板上安装程序包。

### 启动模式

> launch mode

语音交互程序支持 4 种启动模式，接入者可以根据实际情况选用其中的一种。

注意：启动模式是互斥的。同一时间，只可以允许 4 种启动模式中的一种。

- play

  在此启动模式下，仅『音频播放』相关的功能可以使用。

  语音识别、语音对话等功能不可用。

  文本播放功能在此模式下『不』可用。

- asr

  在此启动模式下，仅『语音识别』相关的功能可以使用。

  音频播放、语音对话等功能不可用。

  文本播放功能在此模式下『不』可用。

- audio

  在此启动模式下，可以同时使用『语音识别』和『音频播放』功能，包括『文本播放』功能。

  语音对话功能不可用。

- chat

  在此启动模式下，可以同时使用『语音识别』、『音频播放』和『语音对话』功能。

启动命令统一为：

```
ros2 launch lyre <mode>.launch.py
```

例如，启动 `chat` 模式的命令如下：

```
ros2 launch lyre chat.launch.py
```

每种启动模式支持的 话题 和 服务，可以参考 `lyre` 包中关于启动模式的[描述](../README.md#启动模式)。

## 用例说明

### 测试机型

需要同时测试 Lite 版本和非 Lite 版本的机器。

- `Lite` 版本 —— 测试语音程序包在 x86 主控板上的运行情况。
- 非 `Lite` 版本 —— 测试语音程序包在 Orin 主控板上的运行情况。

### 启动模式

需要同时测试四种启动模式下的工作情况。

### 测试用例编号说明

<platform>-<launch_mode>-<category>-<case>

- platform

  - 0: x86
  - 1: orin

- launch_mode

  - 0: 运行前 
  - 1: play 模式
  - 2: asr 模式
  - 3: audio 模式
  - 4: chat 模式

- category

  - 0: 一般功能
  - 1: 播放功能
  - 2: 识音功能
  - 3: 合成功能
  - 4: 语音功能

- case

  用例号。

例如：

- 用例 `1-0-1-1` 可能表示： Orin 平台的程序包的名称不符合规范。
- 用例 `0-1-2-0` 可能表示： 在 x86 启动 play 模式时，设备不应该提供识音功能。
